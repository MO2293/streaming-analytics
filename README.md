# Streaming Engagement Analytics

An end-to-end analytics project for a mock movie & TV streaming platform. Synthetic data is generated with Python, cleaned and transformed into analysis-ready datasets, modelled in SQL and Excel, and visualised in an interactive Power BI dashboard.

## Tech Stack

Python (pandas, numpy) · SQL · Excel · Power BI (DAX, Power Query)

## Repository Structure

```text
data/
  raw/          # Synthetic source data (titles, users, watch_history)
  processed/    # Cleaned, analysis-ready watch history
src/
  generate_data.py        # Generates the synthetic datasets
  clean_watch_history.py  # Cleans & transforms watch history
  run_pipeline.py         # Runs generate + clean end to end
sql/
  structured_views.sql    # Reusable analytical views (structured datasets)
  exploratory_queries.sql # Ad-hoc exploratory analysis queries
excel/
  engagement_trends.xlsx  # Plan/country/genre pivots + ARPU what-if model
docs/
  metric_definitions.md   # Metric definitions & reporting assumptions
PowerBI/
  Streaming_Insights.pbix # Interactive dashboard (3 pages)
```

## How to Run

```bash
python -m venv venv
venv\Scripts\activate        # Windows
pip install -r requirements.txt
python src/run_pipeline.py   # generates raw data + cleaned dataset
```

Then open `PowerBI/Streaming_Insights.pbix` to explore the dashboard, and
`excel/engagement_trends.xlsx` for the plan/country/genre analysis and ARPU model.

## Dashboard Pages

The report generated by Power BI consists of different pages, with each page giving answers to different business questions.

### 1. Overview

**Goal:** The general state of the platform.

It gives a snapshot of the platform's performance through the provision of KPIs suitable for the executives and trends, such as:

- **Total Watch Time** – total watched minutes in the chosen timeframe.
- **Active Users** – number of distinct users with one or more viewing event.
- **Avg Watch Time per User** – the user engagement calculated by the user who spent time watching.
- **Completion Rate** – the ratio of watching sessions that get to about ~70% of the title.

The most important visuals are:

- **Total Watch Time Time-series chart** which portrays the change in the total watch time over a period (for example, daily or monthly).
- **Watch Time by Subscription Plan** to show the differences in engagement among Basic/Standard/Premium.
- **Watch Time by Country** to point out which countries have the most engaged viewers.

Stakeholders can highlight particular segments by using the interactive slicers (for instance, **country**, **subscription plan**, **date range**).

---

### 2. Content Performance

**Goal:** to discover what content causes the most interaction.

The page is centered on the catalog itself, including genres, ratings, and individual aforementioned titles:

- **Watch Time by Genre** to determine which genres have the most viewings.
- **Watch Time by Content Rating** (G, PG, PG-13, R, TV-14, TV-MA) to evaluate the performance of age ratings comparatively.
- **Top Titles table** that displays for each title:
  - Title, main genre, and, format (Movie/Series)
  - Total Watch Time
  - Avg Minutes per Event
  - Completion Rate

There are filters like **Format (Movie vs Series)**, **Original vs Non-Original**, and **Genre** that facilitate the exploration of inquiries such as:

- “Do originals sell better than licensed titles?”
- “What genres are most favored for series vs movies?”
- “What features do our top 20 titles share?”

---

### 3. User Behavior

**Goal:** Understand how and when users watch content.

This page focuses on user engagement patterns across devices, subscription tiers, and time of day.

Key visuals include:

- **Watch Time by Device Type** to compare viewing across TV, Mobile, Web, and Tablet.
- **Watch Time by Subscription Plan** to analyze how engagement differs across Basic, Standard, and Premium users.
- **Watch Time by Hour of Day** to identify when users are most active on the platform.
- **Hour vs Day-of-Week heatmap** *(if included)* to highlight peak viewing windows across the week.

This page helps answer questions such as:

- Which devices drive the most watch time?
- When are users most active during the day?
- Do higher subscription tiers show stronger engagement patterns?
- How does viewing behavior vary across user segments?

Interactive filters such as **country**, **subscription plan**, and **content format** allow behavior patterns to be explored across different audience groups.
